## Maintainer Responsibilities

### All Maintainers

- Review and merge pull requests in their areas
- Respond to issues and questions
- Participate in design discussions
- Help onboard new contributors
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md)

### Core Maintainers (Additional)

- Make architectural decisions via ADRs
- Coordinate releases across components
- Manage project governance
- Resolve disputes between component maintainers
- Approve new maintainers

## Becoming a Maintainer

### Path to Component Maintainer

1. **Consistent contribution**: Multiple accepted PRs over 3+ months
2. **Quality**: Contributions are high quality with minimal revision needed
3. **Review**: Provides thoughtful code reviews for others
4. **Community**: Helpful in issues and discussions
5. **Nomination**: Nominated by existing maintainer
6. **Approval**: Approved by core maintainers

### Path to Core Maintainer

1. **Component maintainer**: Successful track record as component maintainer
2. **Cross-component work**: Contributions spanning multiple repositories
3. **Architecture**: Participation in design discussions and ADRs
4. **Leadership**: Demonstrated technical leadership
5. **Nomination**: Nominated by existing core maintainer
6. **Approval**: Unanimous approval by core maintainers
